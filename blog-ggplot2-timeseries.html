Visualizing Ecological Time Series with R and ggplot2

Lessons from plotting multi-year plankton abundance data with the grammar of graphics.

One of the most rewarding parts of my postdoc work in marine ecosystem modeling was the visualization stage -- the moment when months of simulation data finally resolves into a clear picture. For this, I relied almost entirely on R's ggplot2 package. Coming from a background where I had cobbled together matplotlib figures with endless ax.set_xlim() calls, ggplot2's declarative approach felt like a revelation. In this post, I want to walk through how I built publication-quality time series plots for plankton abundance data, and why the grammar of graphics philosophy makes such a difference for exploratory scientific visualization.

The Grammar of Graphics Philosophy

ggplot2 is built on Leland Wilkinson's Grammar of Graphics, which decomposes every plot into independent, composable components: data, aesthetic mappings (which variables map to which visual properties), geometric objects (points, lines, bars), scales (how data values translate to visual values), facets (subplots by category), and themes (visual styling). Rather than calling a monolithic "line plot" function with dozens of arguments, you assemble a plot from these layers using the + operator.

This matters in practice because scientific visualization is inherently iterative. You start with a rough plot, notice something interesting, and want to quickly add a smoothing line, split by species, or change the color scale. With ggplot2, each of those changes is a single line addition rather than a structural rewrite of your plotting code.

# The core idea: a plot is data + aesthetics + geometry
library(ggplot2)

# These three lines define a complete plot
ggplot(data = plankton_df,           # 1. Data
       aes(x = date, y = abundance,  # 2. Aesthetic mapping
           color = species)) +
  geom_line()                        # 3. Geometry

Everything else -- scales, labels, facets, themes -- is added incrementally on top of this foundation.

Building a Time Series Plot Step by Step

Let me walk through a realistic example. During my postdoc, I worked with simulated phytoplankton abundance data from the ICON-O/HAMOCC biogeochemical model. The dataset contained daily concentrations of several plankton functional types across multiple years. Here is how I would build up the visualization layer by layer:

library(tidyverse)
library(lubridate)

# Load and prepare the data
plankton_df <- read_csv("hamocc_plankton_output.csv") %>%
  mutate(date = ymd(date)) %>%
  filter(date >= "2015-01-01", date <= "2020-12-31")

# Step 1: Basic time series -- just lines
p <- ggplot(plankton_df, aes(x = date, y = abundance, color = species)) +
  geom_line(alpha = 0.7, linewidth = 0.5)

# Step 2: Proper date axis formatting
p <- p +
  scale_x_date(
    date_breaks = "1 year",
    date_labels = "%Y",
    expand = expansion(mult = 0.02)
  ) +
  scale_y_continuous(labels = scales::scientific)

# Step 3: Clean theme and labels
p <- p +
  labs(
    title = "Phytoplankton Abundance in the North Sea (2015-2020)",
    subtitle = "Daily concentrations from HAMOCC biogeochemical model output",
    x = NULL,
    y = expression(paste("Abundance (cells ", L^{-1}, ")")),
    color = "Functional Type"
  ) +
  theme_minimal(base_size = 13) +
  theme(
    legend.position = "bottom",
    panel.grid.minor = element_blank(),
    plot.title = element_text(face = "bold")
  )

# Step 4: Split into panels by species for clarity
p <- p +
  facet_wrap(~ species, scales = "free_y", ncol = 1)

ggsave("plankton_timeseries.png", p, width = 10, height = 12, dpi = 300)

Each step adds exactly one conceptual layer. The facet_wrap(~ species, scales = "free_y") call is particularly important for ecological data: different plankton functional types can vary by orders of magnitude in abundance, so allowing each panel to have its own y-axis scale prevents the dominant species from squashing the others into flat lines. The ncol = 1 argument stacks the panels vertically, which makes it easy to compare the timing of blooms across species even when their magnitudes differ.

For presentations, I often added a vertical reference line to mark specific events, like the onset of a simulated heatwave:

# Add an annotation for a marine heatwave event
p + geom_vline(
    xintercept = as.Date("2018-07-01"),
    linetype = "dashed", color = "firebrick", linewidth = 0.8
  ) +
  annotate("text", x = as.Date("2018-08-15"), y = Inf,
           label = "MHW onset", vjust = 1.5, color = "firebrick",
           fontface = "italic", size = 3.5)

Smoothing and Trend Lines

Raw daily plankton data is noisy. Strong seasonal cycles, interannual variability, and model spinup artifacts can make it hard to see the underlying trend. ggplot2 makes it trivial to overlay a smoothed trend line with geom_smooth():

# LOESS smoothing -- ideal for non-linear ecological trends
ggplot(plankton_df %>% filter(species == "Diatoms"),
       aes(x = date, y = abundance)) +
  geom_line(color = "steelblue", alpha = 0.4, linewidth = 0.3) +
  geom_smooth(
    method = "loess",
    span = 0.15,         # Controls smoothness: lower = more local
    color = "darkblue",
    fill = "lightblue",
    alpha = 0.3
  ) +
  scale_x_date(date_breaks = "1 year", date_labels = "%Y") +
  labs(
    title = "Diatom Abundance with LOESS Trend",
    x = NULL,
    y = expression(paste("Abundance (cells ", L^{-1}, ")"))
  ) +
  theme_minimal(base_size = 13)

I chose LOESS (Locally Estimated Scatterplot Smoothing) over a simple linear regression for a specific reason: ecological time series are almost never linear. Plankton populations exhibit strong seasonal blooms (typically a spring bloom driven by light and nutrient availability, sometimes a secondary autumn bloom), and these periodic patterns would be completely missed by a straight line. LOESS fits a local polynomial at each point, weighted by neighboring observations, so it naturally captures the seasonal envelope while smoothing out day-to-day noise.

The span parameter controls the trade-off between smoothness and fidelity. A span of 0.15 means each local fit uses roughly 15% of the data, which for a 6-year daily dataset corresponds to about 11 months -- enough to capture the seasonal pattern without overfitting to individual weeks. If you increase the span to 0.5 or higher, the curve smooths over the seasons entirely and shows only the multi-year trend. Choosing the right span is a scientific judgment call, not just a statistical one: it depends on what temporal scale you care about.

For comparing smoothed trends across multiple species simultaneously, I combined LOESS with faceting:

# Compare seasonal patterns across functional types
ggplot(plankton_df, aes(x = date, y = abundance)) +
  geom_line(aes(color = species), alpha = 0.3, linewidth = 0.3) +
  geom_smooth(aes(color = species, fill = species),
              method = "loess", span = 0.1, alpha = 0.2) +
  facet_wrap(~ species, scales = "free_y", ncol = 2) +
  scale_x_date(date_breaks = "2 years", date_labels = "%Y") +
  theme_minimal(base_size = 12) +
  theme(legend.position = "none") +
  labs(title = "Seasonal Patterns by Plankton Functional Type",
       x = NULL,
       y = expression(paste("Abundance (cells ", L^{-1}, ")")))

This kind of multi-panel exploratory figure, which would take considerable effort to build from scratch in most plotting libraries, comes together in ggplot2 with remarkable ease. It is one of the reasons R remains my go-to for statistical visualization, even when the rest of my pipeline is in Python.

Interview Quick Review

What is ggplot2?

ggplot2 is an R package for data visualization based on the Grammar of Graphics, where plots are built by composing independent layers -- data, aesthetic mappings, geometric objects, scales, facets, and themes.

When would you use it?

For any statistical or scientific visualization in R, especially exploratory plots where you need to iterate quickly -- changing color mappings, adding facets, overlaying trend lines -- without rewriting your code from scratch.

Key function to remember:

ggplot(df, aes(x, y)) + geom_line()

Common interview question:

"What is the grammar of graphics?"

It is a data visualization philosophy where plots are constructed from composable, independent layers: data (the dataset), aesthetics (mappings from variables to visual properties like position, color, size), geometries (the visual marks -- points, lines, bars), scales (how data values map to visual values), facets (subplots by categorical variables), and themes (non-data visual styling). This decomposition makes plots modular and easy to modify incrementally.

One thing to remember:

The + operator in ggplot2 is not just syntax sugar -- it reflects the core design principle that every visual element is an independent layer. You can save a base plot to a variable and add layers conditionally, making it easy to build reusable plotting functions.
