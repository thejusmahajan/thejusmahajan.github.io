Gene Expression Analysis with DESeq2: From Counts to Insights

A walkthrough of differential gene expression analysis using DESeq2, from raw count matrices to volcano plots.

The RNA-seq Analysis Pipeline

RNA-seq has become the standard method for measuring genome-wide gene expression. The typical analysis pipeline looks like this:

  1. Raw reads — FASTQ files from the sequencer
  2. Quality control — adapter trimming, quality filtering (FastQC, Trimmomatic)
  3. Alignment — mapping reads to a reference genome (STAR, HISAT2)
  4. Quantification — counting reads per gene (featureCounts, HTSeq)
  5. Count matrix — a genes-by-samples matrix of integer counts
  6. Differential expression — statistical testing for differences between conditions

DESeq2, developed by Love, Huber, and Anders (2014), handles that last step. It takes the raw count matrix as input — not TPM, not FPKM, not normalized values. This is important because DESeq2 performs its own normalization internally, and feeding it pre-normalized data will produce incorrect results.

Coming from a physics background where data is often continuous and approximately normal, working with count data required a genuine shift in thinking. Counts are discrete, non-negative, and their variance scales with their mean in ways that normal distributions cannot capture. DESeq2 is built around a statistical framework that respects these properties.

Setting Up DESeq2

The workflow in R is remarkably concise for how much statistical machinery is operating under the hood. You need two inputs: a count matrix (genes as rows, samples as columns) and a metadata table describing the experimental conditions for each sample.

library(DESeq2)

# counts: integer matrix of raw read counts (genes x samples)
# metadata: data frame with sample info, must include the design variable
# Rows of metadata must match columns of counts

dds <- DESeqDataSetFromMatrix(
  countData = counts,
  colData   = metadata,
  design    = ~ condition
)

# Optional: pre-filter low-count genes to speed up computation
keep <- rowSums(counts(dds)) >= 10
dds  <- dds[keep, ]

# Run the full DESeq2 pipeline in a single command
# This performs: estimation of size factors, estimation of
# dispersions, and negative binomial GLM fitting + Wald test
dds <- DESeq(dds)

# Extract results for a specific contrast
res <- results(dds, contrast = c("condition", "treated", "control"))

# View the top differentially expressed genes
res_ordered <- res[order(res$padj), ]
head(res_ordered)

The design = ~ condition formula tells DESeq2 that you want to test for differences between levels of the "condition" variable. For more complex designs (e.g., including batch effects), you can extend this: design = ~ batch + condition. The variable of interest should always be the last term in the formula.

Understanding the Statistics

DESeq2 models the read counts using a negative binomial (NB) distribution, which is parameterized by a mean and a dispersion. This is the key statistical choice that makes it appropriate for RNA-seq data.

Why not a simpler model? If read counts followed a Poisson distribution, the variance would equal the mean. In practice, RNA-seq data is overdispersed: the variance consistently exceeds the mean, due to biological variability between replicates. The negative binomial distribution adds a dispersion parameter that captures this extra variability. You can think of it as: Poisson handles the technical (sampling) noise, and the dispersion parameter handles the biological noise on top of that.

DESeq2 performs several sophisticated steps internally:

  • Size factor estimation — normalizes for differences in sequencing depth between samples using the median-of-ratios method. This is more robust than simple total-count normalization because it is not skewed by highly expressed genes.
  • Dispersion estimation — estimates a gene-wise dispersion, then shrinks these estimates toward a fitted trend line. This "information sharing" across genes stabilizes estimates for genes with few replicates, which is almost always the case in practice.
  • Log2 fold change shrinkage — optionally shrinks LFC estimates for genes with low counts or high dispersion, producing more reliable rankings.

The results table contains several key columns:

# Key columns in the results table:
# baseMean        - average normalized count across all samples
# log2FoldChange  - estimated effect size (treated vs control)
# lfcSE           - standard error of the LFC estimate
# stat            - Wald test statistic (LFC / lfcSE)
# pvalue          - raw p-value from the Wald test
# padj            - Benjamini-Hochberg adjusted p-value (FDR)

# Summary of results at FDR < 0.05
summary(res, alpha = 0.05)

The padj column is the one you should use for determining significance. It controls the false discovery rate (FDR) using the Benjamini-Hochberg procedure. A typical threshold is padj < 0.05, meaning you accept that up to 5% of your "significant" genes may be false positives.

Visualization

Good visualizations are essential for both quality control and communicating results. DESeq2 and ggplot2 together provide a powerful toolkit:

MA plot — shows the relationship between mean expression and fold change. Most genes should cluster around LFC = 0, with significant genes highlighted:

# Built-in MA plot
plotMA(res, ylim = c(-5, 5), alpha = 0.05)

# Points in blue are significant at padj < 0.05
# The X-axis is mean normalized counts (log scale)
# The Y-axis is log2 fold change

Volcano plot — perhaps the most iconic RNA-seq visualization. It plots statistical significance against fold change magnitude, making it easy to identify genes that are both highly significant and strongly differentially expressed:

library(ggplot2)

res_df <- as.data.frame(res)
res_df$significant <- ifelse(
  res_df$padj < 0.05 & abs(res_df$log2FoldChange) > 1,
  "Significant", "Not significant"
)

ggplot(res_df, aes(x = log2FoldChange, y = -log10(padj),
                   color = significant)) +
  geom_point(alpha = 0.6, size = 1.5) +
  scale_color_manual(values = c("grey60", "firebrick3")) +
  geom_vline(xintercept = c(-1, 1), linetype = "dashed", color = "grey40") +
  geom_hline(yintercept = -log10(0.05), linetype = "dashed", color = "grey40") +
  labs(x = "Log2 Fold Change", y = "-Log10 Adjusted P-value",
       title = "Volcano Plot: Treated vs Control") +
  theme_minimal(base_size = 14) +
  theme(legend.position = "bottom")

Heatmap of top DE genes — shows expression patterns across samples for the most significant genes, useful for spotting sample clustering and expression patterns:

library(pheatmap)

# Use variance-stabilized counts for visualization
vsd <- vst(dds, blind = FALSE)

# Select top 30 DE genes by adjusted p-value
top_genes <- head(rownames(res_ordered), 30)
mat <- assay(vsd)[top_genes, ]

# Z-score normalize rows for visualization
mat_z <- t(scale(t(mat)))

pheatmap(mat_z,
         annotation_col = metadata["condition"],
         clustering_distance_rows = "correlation",
         clustering_distance_cols = "correlation",
         fontsize_row = 8,
         main = "Top 30 Differentially Expressed Genes")

One important note: always use variance-stabilized or rlog-transformed counts for visualization and clustering — never raw counts. The vst() transformation stabilizes variance across the range of mean values, making distances between samples meaningful for PCA, heatmaps, and other exploratory analyses.

Interview Quick Review

What is DESeq2?

A Bioconductor R package for differential gene expression analysis of RNA-seq count data. It uses a negative binomial generalized linear model with shrinkage estimation of dispersions and fold changes.

When would you use it?

When you have a raw count matrix from an RNA-seq experiment and need to identify genes that are differentially expressed between experimental conditions (e.g., treated vs. control, disease vs. healthy), while properly controlling for multiple testing.

Key function to remember:

dds <- DESeq(dds)
res <- results(dds)

Common interview question:

"Why does DESeq2 use a negative binomial distribution instead of a normal distribution for RNA-seq data?"

RNA-seq count data is discrete and overdispersed — the variance exceeds the mean, which violates the Poisson assumption (variance = mean) and certainly cannot be captured by a normal distribution. The negative binomial distribution includes an extra dispersion parameter that models this biological overdispersion. DESeq2 further stabilizes dispersion estimates by sharing information across genes, which is critical when sample sizes are small.

One thing to remember:

Always use raw counts as input to DESeq2, never TPM or FPKM. The package performs its own normalization (median-of-ratios method), and pre-normalized data will produce incorrect statistical results.
