Command Line Core: Navigating NCBI & Biopython

Essential reference notes on using advanced bash scripting, E-utilities, and Biopython for programmatic retrieval and rapid parsing of genomic data from the NCBI databases.

1. The Power of E-utilities (Bash)

Relying on a GUI web browser to download FASTA files from NCBI is a cardinal sin in bioinformatics pipeline engineering. To build reproducible workflows (like my Hepatitis virus pipeline), programmatic access via Entrez Direct (E-utilities) is mandatory.

Querying and Downloading via CLI

The core pipeline uses esearch to query the database and pipes the IDs to efetch to download the physical data. Here are my standard snippets:

# Search the nucleotide database for Hepatitis Delta Virus complete genomes and download in FASTA format
esearch -db nucleotide -query "Hepatitis Delta Virus[Organism] AND complete genome[Title]" | \
  efetch -format fasta > hdv_genomes.fasta

# Extracting just the accession numbers instead for a downstream Nextflow channel
esearch -db nucleotide -query "txid12461[Organism:exp]" | efetch -format acc > accessions.txt

2. Parsing with Biopython

Once raw data is downloaded, bash commands like grep and awk are fast but brittle when handling complex multiline fasta files. This is where Python's Bio.SeqIO becomes the heavy lifter.

Filtering Sequences

A frequent task is filtering out sequences that are too short (likely fragments) or contain unknown bases ('N').

from Bio import SeqIO

def filter_quality_sequences(input_fasta, output_fasta, min_length=1500):
    valid_records = []
    
    # Parse the raw fasta
    for record in SeqIO.parse(input_fasta, "fasta"):
        # Check length constraint and ensure no 'N' (unknown) nucleotides exist
        if len(record.seq) >= min_length and "N" not in record.seq.upper():
            valid_records.append(record)
            
    # Write the high quality records back to a new file
    SeqIO.write(valid_records, output_fasta, "fasta")
    print(f"Filtered {len(valid_records)} high-quality sequences.")

filter_quality_sequences("hdv_genomes.fasta", "hdv_filtered.fasta")

Memory Note

If the input fasta is massive (multi-gigabyte like human genome chunks), you cannot hold valid_records in memory as a list. You must yield them via a generator and write sequentially, or use SeqIO.index() for dictionary-like random access on disk.
