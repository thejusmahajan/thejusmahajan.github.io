Understanding BLAST: Sequence Alignment from First Principles

Breaking down the most-used bioinformatics tool — how BLAST finds similar sequences and what the E-value actually means.

Why Sequence Alignment Matters

One of the foundational principles of molecular biology is that sequence determines structure, and structure determines function. If two proteins share significant sequence similarity, they almost certainly share a common ancestor, and there is a strong likelihood they perform related functions. This is homology inference, and it is the backbone of a remarkable amount of modern biology.

In practice, this means that when you sequence a new gene or protein, the first thing you do is search it against known databases to find similar sequences. This approach drives functional annotation of newly sequenced genomes, identification of conserved domains, evolutionary and phylogenetic analysis, and clinical interpretation of genetic variants. The tool that makes all of this computationally feasible is BLAST — the Basic Local Alignment Search Tool, first published by Altschul et al. in 1990 and still one of the most cited papers in all of science.

How BLAST Works (Simplified)

The "gold standard" for local sequence alignment is the Smith-Waterman algorithm, which uses dynamic programming to find the optimal local alignment between two sequences. It is guaranteed to find the best alignment, but it runs in O(mn) time where m and n are the sequence lengths. Searching a query against millions of database sequences this way would take an impractical amount of time.

BLAST solves this with a heuristic three-stage approach that sacrifices a small amount of sensitivity for massive gains in speed:

Stage 1: Seeding. The query sequence is broken into overlapping words of length k (default k=11 for nucleotide, k=3 for protein). For protein searches, BLAST also generates "neighborhood words" — words that score above a threshold when compared to the query word using a substitution matrix like BLOSUM62. These seeds form the initial search targets.

Stage 2: Extension. When a seed matches a location in the database, BLAST extends the alignment in both directions from the seed. In modern BLAST (BLAST+), this uses a two-hit approach: extension is only triggered when two seeds on the same diagonal fall within a certain distance of each other, which dramatically reduces false extensions. Extension continues as long as the cumulative alignment score does not drop more than a threshold X below the best score seen so far.

Stage 3: Evaluation. High-scoring segment pairs (HSPs) that survive extension are evaluated statistically. BLAST computes a raw alignment score, converts it to a bit score (normalized and database-independent), and calculates an E-value based on the Karlin-Altschul statistics for local alignment scores. The E-value depends on the database size, so the same alignment will have different E-values when searched against databases of different sizes.

Reading BLAST Output

Understanding BLAST output is critical. The three numbers you should always look at are:

  • E-value (Expect value): The number of alignments with this score or better that you would expect to see by chance in a database of this size. An E-value of 1e-50 means you would expect to see a hit this good by chance once in 1050 searches. Lower is better. A common threshold for "significant" homology is E < 1e-5, though this depends on the application.
  • Bit score: A normalized version of the raw alignment score that is independent of the scoring system and database size. Higher is better. Bit scores allow comparison across different searches.
  • Percent identity: The fraction of aligned positions that are identical. Useful but can be misleading without considering alignment length — a 100% identity over 20 residues is far less meaningful than 60% identity over 300 residues.

A typical command-line BLAST search looks like this:

# Nucleotide BLAST against the nt database
# -outfmt 6 gives tabular output (easy to parse programmatically)
blastn -query my_seq.fasta -db nt -out results.txt -evalue 1e-5 -outfmt 6

# Tabular output columns (outfmt 6 default):
# qseqid sseqid pident length mismatch gapopen
# qstart qend sstart send evalue bitscore

The tabular format (-outfmt 6) is particularly useful for downstream processing in R or Python, since each hit is a single tab-delimited row. For quick exploration, the default output (-outfmt 0) provides a more human-readable report with pairwise alignments.

BLAST Flavors

BLAST comes in several variants, each designed for a different combination of query and database sequence types:

Program Query Database Use case
blastn Nucleotide Nucleotide Finding similar DNA/RNA sequences
blastp Protein Protein Finding homologous proteins
blastx Nucleotide (translated) Protein Searching a DNA sequence against protein DB; useful for ESTs or novel transcripts
tblastn Protein Nucleotide (translated) Searching a protein against a nucleotide DB; finding unannotated coding regions
tblastx Nucleotide (translated) Nucleotide (translated) Comparing translations of both; computationally expensive, used for distantly related organisms

The general rule of thumb: protein-level comparisons are more sensitive than nucleotide comparisons for detecting distant homology, because the genetic code is degenerate — many nucleotide changes are synonymous at the protein level. If you are looking for conserved function across distantly related organisms, use blastp or blastx rather than blastn.

Interview Quick Review

What is BLAST?

The Basic Local Alignment Search Tool — a heuristic algorithm that finds regions of local similarity between a query sequence and a database of sequences, using a seed-and-extend strategy that is orders of magnitude faster than optimal dynamic programming methods like Smith-Waterman.

When would you use it?

Whenever you need to identify what a sequence is, find homologous genes or proteins in other organisms, annotate newly assembled genomes, or assess whether a variant falls in a conserved region. It is typically the first analysis step after sequencing.

Key function to remember:

blastn -query seq.fa -db nt -evalue 1e-5 -outfmt 6

Common interview question:

"What does the E-value in BLAST output mean?"

The E-value (Expect value) is the expected number of alignments with that score or better that would occur by chance in a database of that size. It depends on the query length, database size, and scoring parameters. A lower E-value indicates a more statistically significant hit. An E-value of 1e-5 means you would expect to see one hit this good by chance in every 100,000 random searches.

One thing to remember:

Protein-level comparisons (blastp, blastx) are more sensitive than nucleotide comparisons (blastn) for detecting distant homology, because synonymous codon changes erode nucleotide similarity while preserving amino acid identity.
